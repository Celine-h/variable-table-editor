# 设计文档

## 技术选型

- 项目搭建、基础功能以及状态管理技术栈：`Umi4 + React + Ts + Mobx + Sass + Antd `
- 单元测试：`Jest + React-Test-Library`

## 架构设计

- `UI层`：入口文件`index.ts`；使用antd-table封装`EditableTable`，实现表格单选功能。
- `状态层`：`tableStore`管理表格数据、选中行状态、文本内容、错误提示状态。
- `工具类`：输入内容检验类/导入导出格式转换类

## 设计要点

1. **状态管理**  
   通过mobx`tableStore`管理表格数据、选中行状态、文本内容、错误提示状态 ,保证单一数据源。

2. **导入导出格式转换/校验工具**  
    导入：解析函数`parseImportText`解析导入内容格式；全量解析，成功通过后插入表格，否则表格内容不变并提示具体错误原因，校验规则：重复校验/格式校验/类型不匹配校。  
   导出：处理表格源数据，生成符合VAR...END_VAR格式的数据，展示在输入框。


## 单元测试

- `可用性测试`:核心按钮和页面基本的交互可用
- `功能性测试`:行增删、单元格名称唯一性，类型切换默认值填充、导入/导出功能测试
